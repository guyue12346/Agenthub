# AgentHub Android

AgentHub Android 是项目的移动端初版，目标是让用户在手机上访问消息、会话和 Agent 协作能力。它不是把桌面四栏工作台直接压缩进 WebView，而是围绕移动端使用习惯重新组织入口和页面层级。

## 产品定位

移动端优先承载高频协作场景：查看会话、进入聊天、跟进 Agent 任务、访问项目消息和保持与正式后端的数据同步。复杂的工作空间管理、资产配置和后台监控仍以 Web 工作台为主。

## 关键能力

- 移动端消息列表与会话详情，适配手机上的单列导航。
- 原生连接设置，支持访问不同部署环境。
- 复用正式后端账号、会话、消息、实时通信和 Hub 数据。
- 在移动端保留 Agent 协作入口，而不是只做静态通知查看。

## 设计取舍

Android 端以轻量、可维护为优先。原生层负责 WebView 容器、连接配置、本地存储和容错；业务页面由移动端 Web 路由承载，避免移动端和 Web 端出现两套割裂的业务实现。
