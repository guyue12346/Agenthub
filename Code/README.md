# AgentHub 工程目录

`Code` 是 AgentHub 的正式工程主体，承载前端、后端、共享协议、桌面端、移动端和 Runner 能力。它对应的是项目当前实现，而不是早期原型或演示脚本。

## 工程分层

| 目录 | 说明 |
| --- | --- |
| `shared` | 共享类型、消息块、运行事件、协议定义和跨端状态约定 |
| `server` | 后端服务，负责账号、会话、消息、Agent、Hub、Workspace、Runtime、日志和监控 |
| `web` | Web 工作台，包含消息、联系人、工作空间、Hub、监控和详情面板 |
| `desktop` | 桌面端外壳，面向局域网或私有部署场景提供独立入口 |
| `mobile` | Android 初版，围绕移动端消息和会话体验重新组织界面 |
| `runner` | Code Agent 的执行边界，用于承载真实代码任务和隔离运行环境 |

## 设计重点

### 协议先行

`shared` 中沉淀消息、事件和状态协议，让前端、后端和 Runner 之间有稳定的协作边界。Agent 的输出不是简单字符串，而是可以承载工具调用、运行状态、产物预览和任务结果的结构化消息。

### 工作台一体化

Web 端不是单页聊天 demo，而是以四区工作台承载会话、联系人、工作空间、Hub 资产和详情面板。用户可以在同一个界面中完成沟通、调用 Agent、查看任务状态和管理资产。

### Runtime 可观测

后端把 Agent 执行过程拆成可追踪的 Runtime Event、Agent Run、Tool Run、LLM 调用日志和系统日志。这样可以复盘任务链路，也方便在后台监控中定位失败原因。

### Runner 隔离

Code Agent 的真实执行通过 Runner 边界接入，避免把代码任务直接混进普通聊天逻辑。工作空间、运行日志、命令脱敏和任务串行控制都围绕这个边界设计。

### 多端一致

Web、桌面端和移动端共享正式后端能力，但各自按使用场景组织交互。桌面端强调独立入口，移动端强调消息列表、会话详情和轻量访问。
