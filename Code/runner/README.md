# AgentHub Runner

Runner 是 AgentHub 中承载 Code Agent 真实执行的边界。它让 Agent 可以处理代码任务，同时把工作空间、命令、日志、密钥、资源限制和运行结果从普通聊天流程中隔离出来。

## 角色定位

在 AgentHub 中，聊天负责交互，Runtime 负责任务编排，Runner 负责执行边界。Code Agent 需要读写文件、调用 CLI、生成 Diff 或产物时，应通过 Runner 进入受控工作空间，而不是直接污染服务端进程或用户本机环境。

## 核心亮点

### 工作空间隔离

每个代码任务围绕明确的工作空间运行。Runner 只暴露任务需要访问的目录，避免 Agent 任意读写宿主机文件。

### 日志与事件闭环

Runner 的命令、输出、错误、退出状态和产物结果会回写 Runtime Event、System Log 和任务记录。用户和管理员可以从系统中追踪任务过程，而不是只看到最终一句成功或失败。

### 密钥边界

模型 Key、访问令牌和运行配置不进入用户工作空间。后端展示命令和日志时会做脱敏处理，Runner 只拿到执行所需的最小配置。

### 可替换执行后端

Runner 设计上支持 Docker Runner 和本地 Runner 两类实现。正式路径强调容器隔离和资源约束，本地路径只作为开发调试兜底。

## 与主系统的关系

Runner 不负责产品交互，也不决定 Agent 策略。它是 Runtime 下游的执行层，负责把“Agent 要做的代码任务”变成可记录、可审计、可复盘的真实执行过程。
